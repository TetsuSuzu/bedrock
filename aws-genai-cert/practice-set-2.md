# AWS Certified Generative AI Developer Professional — 追加練習問題 15 問

番号なしの練習問題群。各問: 要旨（日本語）/ 正解 / 解説。

---

## Q1 — RAG 検索評価
理想的検索結果が既知のクエリ群で、検索品質を検証したい。
**正解: D** — Bedrock RAG 評価ジョブを retrieve-only モードでコンテキスト関連性・網羅性を測定。

## Q2 — ドキュメント前処理
スキャン PDF/画像を FM に渡す前の前処理サービスは？
**正解: C** — Amazon Bedrock Data Automation（構造化 JSON/HTML へ変換、GenAI 向け前処理の専用マネージドサービス）。

## Q3 — リージョン障害への回復力
Bedrock リージョン障害時に自動回復、トラフィックを北米内に。
**正解: A** — 北米に限定した Bedrock クロスリージョン推論プロファイル。

## Q4 — 部門別アクセス制御
部門ごとに特定の Bedrock モデル/KB のみ IAM ロールでアクセス。
**正解: B** — IAM Identity Center の権限セットで部門グループを最小権限 IAM ロールにマッピング。

## Q5 — 長い会話のコンテキスト管理
コンテキストが埋まり遅くなる。性能と保持を両立する方法は？
**正解: B** — 古い会話ターンを要約し、要約＋直近メッセージのみコンテキストに含める。

## Q6 — フィードバック収集と人手レビュー
低評価応答を専門家アノテーションに回したい。
**正解: C** — API Gateway＋Lambda で評価取得、DynamoDB 保存、低評価を Amazon A2I に回し人手レビュー。

## Q7 — ベクトルストアの自動同期
S3 文書の追加/更新時に同期。カスタムコード最小限。
**正解: B** — S3 イベント通知で Lambda をトリガーし KB の StartIngestionJob を呼ぶ。

## Q8 — Text-to-SQL の安全性
承認済みビューのみ照会する SQL を安全に生成・実行。
**正解: B** — FM に SQL 生成→Lambda で構文検証→制限付き DB ロールで読み取り専用ビューに実行（多層防御）。

## Q9 — モデルガバナンス文書化
カスタムモデルの用途・制限・学習データソースを文書化。
**正解: A** — SageMaker Model Registry とプログラム可能なモデルカード（Model Cards）。

## Q10 — 大規模 LLM のデプロイ
30B モデルのロードが遅くデプロイ中にヘルスチェック失敗。
**正解: C** — GPU インスタンスにモデル並列ロードで展開し、コンテナ起動タイムアウト（ヘルスチェック猶予）を延長。

## Q11 — RAG の診断指標
正しい回答だが無関係な文書を引用する問題の診断指標は？
**正解: D** — RAG 評価ジョブの引用精度（Citation precision）とコンテキスト関連性。

## Q12 — Confluence 連携
Confluence 文書を KB に取り込み更新に追従。最もシームレス。
**正解: A** — Confluence を Bedrock KB のマネージドデータソースとして構成しスケジュール同期。

## Q13 — 人手承認ステップ
高リスク応答を顧客に返す前に人の承認を入れる。
**正解: B** — Step Functions のタスクトークンで、人が承認するまで実行を一時停止（コールバックパターン）。

## Q14 — データレジデンシー
顧客データが特定リージョンから出ることを禁止しつつ Bedrock 利用。
**正解: C** — VPC エンドポイントでトラフィックを（域内の）VPC 内に留める。※クロスリージョン推論(B)は域外に出るため要件違反。

## Q15 — モデルの動的切替
Lambda 再デプロイなしでモデル切替、段階ロールアウトと容易なロールバック。
**正解: A** — AWS AppConfig にモデルルーティング設定を保存し、デプロイ戦略（段階ロールアウト/自動ロールバック）を使う。

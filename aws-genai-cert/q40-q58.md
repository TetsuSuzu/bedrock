# AWS Certified Generative AI Developer Professional — Q.40〜Q.58

各問: 問題の要旨（日本語）/ 正解 / 解説。

---

## Q.40 — Chain-of-Thought
複雑な多段階数学問題で中間ステップが幻覚し誤答する。推論精度を上げるプロンプト技術は？
**正解: B** — Chain-of-Thought（「ステップbyステップで考えよう」や中間推論を含む few-shot）。問題を論理的手順に分解させ精度向上。

## Q.41 — プライベート接続
Bedrock と S3 間を公衆インターネットを通さず AWS 内に留めたい。最も複雑さが低い構成は？
**正解: B** — Bedrock 用 Interface VPC エンドポイント（PrivateLink）＋ S3 用 Gateway VPC エンドポイント。

## Q.42 — マルチモーダル埋め込み
自然言語で画像検索＋参照画像で類似検索。テキスト/画像で別モデルを保守したくない。
**正解: D** — Amazon Titan Multimodal Embeddings でテキストと画像を同一ベクトル空間に埋め込み、単一インデックスで類似検索。

## Q.43 — SageMaker データ入力モード
1,000 万枚の小さな DICOM。File モードで各エポック開始時に全ダウンロードし遅延。直接ストリーミングしたい。
**正解: A** — FastFile Mode（FUSE で S3 をローカル FS として提示、オンデマンドストリーミング、コード変更ほぼ不要）。

## Q.44 — Guardrails（PII ブロック）
SSN やクレジットカード番号を FM に届く前に検出・ブロック。マネージドでローコードな機能は？
**正解: B** — Bedrock Guardrails の Sensitive Information Filters を PII タイプ＋Block アクションで構成。

## Q.45 — QLoRA
医療画像向けに大規模 VLM を、4-bit 量子化しつつ高精度アダプターを学習する PEFT 手法は？
**正解: D** — QLoRA（4-bit NormalFloat 量子化＋高精度 Low-Rank アダプター）。

## Q.46 — Bedrock Agent のオーケストレーション
ツールを呼ぶか最終応答するかを判断する反復ロジックを担うコンポーネントは？
**正解: B** — Orchestration Prompt（基盤モデルを CoT/ReAct で導き次のステップを決める）。

## Q.47 — Step Functions の Retry/Catch
ThrottlingException はバックオフ付きリトライ、ValidationException は即失敗させたい。
**正解: B** — `ErrorEquals:[ThrottlingException]` の Retry（バックオフ指定）＋ ValidationException 用の別 Catch で Fail 状態へ。

## Q.48 — RetrieveAndGenerate のメタデータフィルタ
department メタデータが Legal の文書だけを対象にしたい。
**正解: C** — vectorSearchConfiguration 内の filter オブジェクトで論理メタデータ式を定義。

## Q.49 — Bedrock Evaluations（RAG）【TWO】
Faithfulness と Relevance を測りたい。必要な構成を 2 つ。
**正解: A・C** — Ground Truth データセット（比較用）＋ Model-based evaluation（ジャッジモデルで採点）。

## Q.50 — Bedrock AgentCore Runtime
複数ターンの状態管理と割り込み処理。AgentCore はどう簡素化する？
**正解: B** — 統一 Session 構造が Trace 情報と Memory を InvokeAgent 呼び出し間で自動永続化し、直近チェックポイントから再開。

## Q.51 — Amazon Q Developer のカスタマイズ
社内独自ライブラリ/ロギングに沿った提案を得る最も運用効率の良い方法は？
**正解: B** — Amazon Q Customization（自社の CodeCommit / GitHub リポジトリに接続）。

## Q.52 — SageMaker 非同期推論
4K 生成で 1 リクエスト約 8 分。呼び出し側を待たせず S3 準備完了を通知したい。
**正解: B** — SageMaker Asynchronous Inference ＋ SNS 通知（大ペイロード・長時間処理向け、最大1時間）。

## Q.53 — Amazon Kendra Retrieve API
インデックス済み PDF から特定パッセージを抽出し Claude 3 のコンテキストに。
**正解: A** — Retrieve API を呼び、ResultItems 配列の Content フィールドからプロンプトを構築。

## Q.54 — Guardrail の A/B テスト
Guardrail v1（厳格）vs v2（緩和）を 50/50 でテストしたい。
**正解: B** — AppConfig または Lambda でセッションごとに guardrailIdentifier / guardrailVersion を割り当て、InvokeModel リクエストで渡す。

## Q.55 — ストリーミング UI のボトルネック
InvokeModelWithResponseStream で TTFT は良好だが UI が数秒ごとにブロック。
**正解: B** — クライアントがチャンクをバッファに蓄積し、イベントごとに文字列全体を重く再レンダリングしている。

## Q.56 — Chain-of-Verification (CoVe)
微妙な不正確さを減らす CoVe の手順は？
**正解: B** — 初期応答→検証質問を生成→検証質問に独立して回答→矛盾を調整して最終検証済み応答を生成。

## Q.57 — リアルタイム配信
ストリーミングトークンを数千の Web クライアントへ低レイテンシ・ポーリングなしで配信。
**正解: B** — InvokeModelWithResponseStream → AWS AppSync GraphQL Subscription → クライアント WebSocket。

## Q.58 — クロスアカウント Bedrock アクセス【TWO】
B のアプリ IAM ロールだけが A のモデルを呼べ、全トラフィックをプライベート AWS バックボーンに。
**正解: A・B** — A 側 Bedrock モデルにリソースベースポリシーで B のロール ARN に InvokeModel を限定付与 ＋ B 側に Interface VPC エンドポイント（PrivateLink）。

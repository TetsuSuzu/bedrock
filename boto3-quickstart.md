# Boto3 1.43.34 ドキュメント — クイックスタート

## クイックスタート

このガイドでは、AWS SDK for Python のインストールまたは更新に必要な手順を詳しく説明します。

SDK は 2 つの主要な Python パッケージで構成されています：**Botocore**（Python SDK と AWS CLI の間で共有される低レベル機能を提供するライブラリ）と **Boto3**（Python SDK 本体を実装するパッケージ）です。

> **注記**
> ドキュメントや開発者は AWS SDK for Python を「Boto3」と呼ぶ傾向があり、本ドキュメントでも同様に表記することがあります。

---

## インストール

Boto3 を使用するには、まず Boto3 本体とその依存関係をインストールする必要があります。

### Python のインストールまたは更新

Boto3 をインストールする前に、Python 3.10 以降を使用していることを確認してください。古いバージョンの Python からプロジェクトを移行する方法については、「Python 3 への移行」を参照してください。

Python の最新バージョンの入手方法については、[公式 Python ドキュメント](https://www.python.org/)を参照してください。

### 仮想環境のセットアップ

サポートされているバージョンの Python をインストールしたら、仮想環境を作成してアクティベートすることでワークスペースを設定してください：

```bash
$ python -m venv .venv
...
$ source .venv/bin/activate
```

> **Windows (PowerShell) の場合**
> ```powershell
> python -m venv .venv
> .\.venv\Scripts\Activate.ps1
> ```

これにより、システムレベルでインストールされたパッケージとの予期しない干渉を避けるための、独立したインストール空間が提供されます。このステップを省略すると、予期しない依存関係の競合や、システム上にインストールされた他のツールとの障害が発生する可能性があります。

### Boto3 のインストール

pip を使って最新の Boto3 リリースをインストールします：

```bash
pip install boto3
```

プロジェクトで特定のバージョンの Boto3 が必要な場合、または特定のバージョンとの互換性に懸念がある場合は、インストール時に制約を指定できます：

```bash
# Boto3 バージョン 1.0 を指定してインストール
pip install boto3==1.0.0

# Boto3 がバージョン 1.15.0 より古くならないようにする
pip install boto3>=1.15.0

# Boto3 がバージョン 1.15.3 より新しくならないようにする
pip install boto3<=1.15.3
```

> **注記**
> Boto3 の最新開発バージョンは GitHub で公開されています。

### AWS Common Runtime (CRT) の使用

Boto3 のデフォルトインストールに加えて、新しい **AWS Common Runtime (CRT)** を含めることを選択できます。AWS CRT は、AWS SDK の新しい基盤となるモジュール式パッケージのコレクションです。各ライブラリは、実装する機能領域において優れたパフォーマンスと最小限のフットプリントを提供します。CRT を使用することで、SDK は可能な限り同じベースコードを共有でき、AWS SDK 全体での一貫性とスループットの最適化が向上します。

AWS CRT が含まれている場合、Boto3 はそれを使用して AWS SDK for Python では通常利用できない機能を組み込みます。

CRT が使用される機能の例：

- Amazon S3 マルチリージョンアクセスポイント
- Amazon S3 オブジェクト整合性
- Amazon EventBridge グローバルエンドポイント
- コンソール認証情報でのログイン

ただし、Boto3 はデフォルトでは AWS CRT を使用しません。Boto3 をインストールする際に `crt` の追加機能を指定することで、CRT の使用を選択できます：

```bash
pip install boto3[crt]
```

CRT を使用しない Boto3 バージョンに戻すには、以下のコマンドを使用します：

```bash
pip uninstall awscrt
```

CRT を再度有効にする必要がある場合は、`boto3[crt]` を再インストールして互換性のあるバージョンの `awscrt` を取得してください：

```bash
pip install boto3[crt]
```

## 設定

Boto3 を使用する前に、IAM コンソールまたは AWS CLI を使って AWS アカウントの認証情報を設定する必要があります。既存のユーザーを選択するか、新しいユーザーを作成できます。

IAM コンソールを使ってユーザーを作成する手順については、「IAM ユーザーの作成」を参照してください。ユーザーが作成されたら、「アクセスキーの管理」を参照して、ユーザー認証に使用するキーの作成と取得方法を確認してください。

AWS CLI がインストールされている場合は、`aws configure` コマンドを使って認証情報ファイルを設定できます：

```bash
aws configure
```

または、認証情報ファイルを手動で作成することもできます。デフォルトの場所は `~/.aws/credentials` です。最低限、認証情報ファイルにはアクセスキーとシークレットアクセスキーを指定する必要があります。以下の例では、デフォルトプロファイルにアカウントのキーとシークレットキーが指定されています：

```ini
[default]
aws_access_key_id = YOUR_ACCESS_KEY
aws_secret_access_key = YOUR_SECRET_KEY
```

> **注意**
> 認証情報ファイル（アクセスキー・シークレットキー）は絶対に Git リポジトリへコミットしないでください。

また、AWS 設定ファイル（デフォルトの場所は `~/.aws/config`）にデフォルトリージョンを追加することもできます：

```ini
[default]
region=us-east-1
```

または、クライアントやリソースを作成する際に `region_name` を渡すこともできます。

これで、デフォルトプロファイルの認証情報と、接続を作成する際に使用するデフォルトリージョンが設定されました。詳細な設定ソースとオプションについては「設定」を参照してください。

## Boto3 の使用

Boto3 を使用するには、まずインポートし、使用するサービスを指定する必要があります：

```python
import boto3

# Amazon S3 を使用する
s3 = boto3.resource('s3')
```

`s3` リソースを取得したら、サービスにリクエストを送信できます。以下のコードは `buckets` コレクションを使用して、すべてのバケット名を出力します：

```python
# バケット名を出力する
for bucket in s3.buckets.all():
    print(bucket.name)
```

バイナリデータのアップロードやダウンロードも可能です。例えば、以下のコードはバケット `amzn-s3-demo-bucket` がすでに存在することを前提として、S3 に新しいファイルをアップロードします：

```python
# 新しいファイルをアップロードする
with open('test.jpg', 'rb') as data:
    s3.Bucket('amzn-s3-demo-bucket').put_object(Key='test.jpg', Body=data)
```

**リソース**と**コレクション**については、以降のセクションで詳しく説明します。
